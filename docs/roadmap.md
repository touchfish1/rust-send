# 产品路线图

> 这份文档是 rust-send 的长期有效计划入口，用来持久化记录当前判断、阶段目标、设计取舍与后续优先级。

## 1. 文档定位

- `docs/task-plan.md`：项目早期的完整实施计划，保留为历史背景与架构参考
- `docs/roadmap.md`：当前有效的产品路线图、阶段目标、决策记录与后续行动

维护原则：

- 新的产品方向、里程碑调整、关键取舍，优先更新这里
- 已完成的阶段保留简要结论，不在这里堆积实现细节
- 具体实现方案可以拆到独立专题文档，但这里始终保留入口和结论

## 2. 产品目标

rust-send 的目标不是只做一个“能传文件”的演示工具，而是做成一个真正可长期使用的跨平台传输应用：

- 打开就能连，尽量少理解成本
- 大文件、弱网、跨平台场景下依然稳定
- 本地直连优先，必要时无感切到中继
- 桌面端与 Web 端体验一致，视觉上轻盈、有产品感
- 安装、升级、回滚、获取新版都足够顺滑

一句话定义：

> rust-send 要做成一个“可信、顺滑、好看”的跨平台文件传输工具。

## 3. 当前状态快照

截至 `v0.1.1`，项目已经具备以下基础能力：

- Tauri 桌面端与 Web 端双端形态已经跑通
- 局域网发现、配对、扫码进入 Web 端等基础链路已具备
- 中继传输、历史记录、设置页、版本信息等基础产品骨架已形成
- GitHub Actions 已支持多平台构建与 GitHub Release 发布
- Release 页面文案已产品化整理
- 应用内更新机制已接入，可通过 GitHub Release 分发更新元数据
- Linux Release 已补充 `AppImage` 产物

当前最需要补齐的，不是“有没有功能”，而是下面三件事：

1. 传输链路是否足够稳
2. 配对与连接体验是否足够顺
3. 产品细节是否足够完整、可信、好用

### 最新进展

#### 2026-06-17

已完成一轮 `v0.2.0` 前置增强，聚焦“历史记录与结果可追溯”：

- 后端开始在完成、失败、取消时写入 `history.json`
- 历史记录补充了传输方式、对端设备 ID、失败原因
- 前端启动时会主动拉取已持久化的历史记录与活动传输
- 历史页已展示更清晰的结束时间、链路信息与失败说明

这意味着历史记录不再只是临时前端状态，而是正式进入可持久追踪阶段。

同日继续推进“设备与连接体验”：

- 已增加“最近设备”持久化列表
- 在线设备与最近设备在侧边栏分层展示
- 最近设备会在设备离线或应用重启后继续保留

这为后续的“可信设备”与“更顺滑重连”打下了直接基础。

同日补齐一轮测试基础设施：

- 新增传输归一化与侧边栏最近设备的前端测试
- 新增测试文档，明确本地运行方式与覆盖范围
- 新增 GitHub Actions `CI` 工作流，在每次 `push` / `pull_request` 时自动运行前端测试、前端构建、Rust 工作区测试与构建

这让后续功能演进不再只依赖手工验证，而是进入了持续集成阶段。

同日继续推进“可信设备”基础能力：

- 已增加可信设备持久化状态
- 侧边栏、聊天页、设置页都可识别并管理可信设备
- 可信设备管理已作为后续自动连接与更安全确认的前置基础

## 4. 近期总方向

未来一段时间优先级按下面顺序推进：

### P0 稳定性

- 大文件传输稳定性
- 中断恢复与失败重试
- Relay 场景的完成度
- 冲突处理、校验、异常提示

### P1 连接体验

- 最近设备与可信设备
- 更顺滑的重新连接
- 更清晰的连接诊断
- 更自然的扫码与配对引导

### P2 产品完成度

- 更完整的历史记录与状态说明
- 更好的升级反馈与版本说明
- 首页、配对页、传输页的动效和微交互
- Release、安装包、帮助文档持续产品化

## 7. 决策记录

### 2026-06：更新策略采用 GitHub Release + Tauri Updater

原因：

- 维护成本低
- 与现有发布流程天然一致
- Windows / macOS 用户可直接获得应用内升级体验
- 适合当前阶段的独立产品节奏

### 2026-06：Linux 同时提供 `AppImage`

原因：

- `AppImage` 对普通 Linux 用户更直接
- 减少发行版差异带来的理解成本
- 更符合“下载即用”的产品预期

### 2026-06：计划必须落库到仓库文档

原因：

- 避免上下文切换后计划丢失
- 方便不同模型、协作者、未来自己快速接手
- 让“当前优先级”和“历史背景”分层清晰

执行约定：

- 新功能开始前，先检查是否需要更新本路线图
- 里程碑完成后，更新当前状态快照与下一阶段目标
- 重大取舍必须补一条决策记录
